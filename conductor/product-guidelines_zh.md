# 產品指導原則：內部 Agent 技能市集

## 1. 視覺識別與 UI 設計
*   **現代繽紛美學：** 介面應採用鮮豔的漸層色（如靛藍、紫色、青色）與毛玻璃效果。
*   **活力動感：** 使用更具現代感的無襯線字體（Sans-serif），取代冰冷的等寬字體。
*   **光暈與層次：** 為元件（如卡片、按鈕）加入柔和的彩色陰影或背光光暈，提升視覺吸引力。

## 2. API 設計與整合
*   **RESTful 基礎：** 遵守標準 HTTP 方法和 JSON 負載，以確保廣泛的相容性。
*   **CLI 友善的使用者體驗：** 優化 API 回應以利終端機使用（例如：簡潔的 JSON、支援常見標頭），使 `curl` 或腳本交互無縫接軌。
*   **功能極簡化：** 保持 API 表面積小巧，專注於核心操作：上傳、下載、搜尋和更新。

## 3. 使用者體驗 (UX) 原則
*   **無摩擦貢獻：** 上傳流程（無論是透過 Web 還是 API）必須極其簡單，只需極少步驟即可分享新技能。
*   **文件即一等公民：** 自動渲染並精美呈現 `SKILL.md` 內容。使用者在 Markdown 檔案中寫什麼，其他人在瀏覽器中就應該看到什麼。

## 4. 開發與工程標準
*   **簡單勝於複雜：** 避免過度工程化。傾向於可讀、可維護的程式碼，讓其他人能輕鬆貢獻。
*   **安全優先心態：** 鑑於內部性質且涉及程式碼處理，確保強大的身份驗證、所有權檢查和秘密掃描整合至核心。
*   **效能優化：** 確保平台在註冊技能數量和同時在線使用者增長時，仍保持快速響應。

## 5. 語氣與聲音
*   **協作與社群驅動：** UI 和文件中的語言應具有鼓勵性和包容性，將平台框架化為知識交換的共享內部公地。
*   **支持性溝通：** 專注於幫助使用者成功分享技能，而不僅僅是執行規則。
